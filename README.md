# Когортный анализ кредитного портфеля и оценка рисков (SQL + Python)

## 📌 Описание проекта
Проект посвящен анализу качества кредитного портфеля банка и поиску дефолтных аномалий. На основе реляционной структуры данных из хранилища (DWH) был проведен когортный анализ клиентов с расчетом ключевой риск-метрики **NPL 90+** (доля кредитов с просрочкой более 90 дней).

## 🛠️ Технологический стек
* **SQL:** сложные JOIN-соединения таблиц, условная агрегация (`CASE WHEN`), группировка по временным периодам.
* **Python (Pandas):** загрузка данных из CSV, трансформация типов данных, сводные таблицы (`.pivot()`).
* **Визуализация:** `Matplotlib`, `Seaborn` (построение тепловых карт / Heatmaps).

## 📊 Ход исследования и результаты
1. С помощью **SQL-запроса** плоские транзакционные данные по ежемесячным платежам были агрегированы в разрезе каналов привлечения и месяцев выдачи кредитов (формирование когорт).
2. Результаты были импортированы в **Python**, где плоская структура была преобразована в когортную матрицу и визуализирована в виде тепловой карты:

### Найденая аномалия и бизнес-инсайт:
* Анализ выявил критический всплеск просрочки в канале привлечения **Partner_Widget** для мартовской когорты клиентов — **NPL 90+ достиг 25%** (при норме по банку в 3-6%). 
* Повторный опасный тренд по этому же каналу зафиксирован в июне (**11%**). 
* Самым стабильным и безопасным каналом является **Mobile_App**, удерживающий уровень просрочки в пределах целевых значений на всем горизонте.

## 💼 Рекомендации для бизнеса
1. **Риск-менеджменту:** Немедленно ужесточить (закрутить) скоринговые правила для трафика, приходящего через партнерский виджет, увеличив требования к кредитной истории и доходу кандидатов.
2. **Маркетингу:** Запустить аудит мартовской и июньской рекламных кампаний на площадке партнера для выявления возможного фрода (намеренного привлечения неплатежеспособной аудитории) или мисселинга.
3. Приостановить бюджеты на масштабирование данного виджета до стабилизации качества входящего потока.
